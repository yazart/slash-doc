# Slash Doc

Slash Doc — расширение для Visual Studio Code, предназначенное для создания и публикации проектной и технической документации. Документы редактируются в блочном редакторе Editor.js и хранятся в рабочей папке проекта в формате JSON.

## Возможности

- древовидная структура страниц с переименованием, вложенностью и drag-and-drop;
- полнотекстовый поиск по документации;
- импорт HTML и Markdown;
- экспорт отдельных страниц в HTML и Markdown;
- сборка статического HTML-сайта с меню, поиском и внутренними ссылками;
- внутренние ссылки между страницами и внешние ссылки, открывающиеся в новой вкладке;
- форматирование текста, таблицы в стиле Confluence, блоки кода и diff с подсветкой синтаксиса;
- Mermaid, Flow Designer, Network Canvas, BPMN Modeler и BPMN Preview;
- аннотирование изображений;
- описание API-эндпоинтов с примерами запросов и моделями TypeScript;
- доска задач в стиле Trello;
- обработка локальных CSV и JSON-файлов пользовательским JavaScript;
- подключаемые Editor.js-аддоны и локальные HTTP API-модули.

## Требования

- Visual Studio Code 1.90 или новее;
- Node.js и npm для разработки и сборки расширения;
- открытая рабочая папка VS Code для хранения документации.

## Использование

1. Откройте папку проекта в VS Code.
2. Откройте представление **Slash Doc** в Activity Bar.
3. Создайте страницу или выполните команду **Slash Doc: Открыть редактор**.
4. Добавляйте блоки через меню Editor.js. Изменения сохраняются автоматически.
5. Используйте иконочные кнопки в панели страниц для импорта документа или сборки HTML-документации.

### Хранение данных

Документация находится внутри рабочей папки:

```text
.slash-doc/
├── sdsettings.json
└── docs/
    ├── menu.json
    └── pages/
        └── <page-id>/
            └── content.json
```

Загруженные файлы блока обработки данных сохраняются рядом с соответствующей страницей. Глобальные пользовательские аддоны находятся в `addons/`, а HTTP API-модули — в `api/` рядом с расширением.

> Пользовательский JavaScript блока обработки файлов выполняется локально с доступом к файлам блока. Запускайте только доверенный код.

### BPMN

- **BPMN Modeler** создаёт и редактирует BPMN 2.0-диаграммы с помощью `bpmn-js 9.0.0`.
- **BPMN Preview** принимает BPMN XML из файла, drag-and-drop или текстового поля.
- XML и SVG сохраняются в данных Editor.js; при HTML-экспорте оба блока становятся статическим SVG.
- Водяной знак bpmn.io является обязательной частью `bpmn-js 9.0.0` и должен оставаться видимым.

## Разработка

```sh
npm install
npm run compile
```

Откройте проект в VS Code, нажмите `F5` и запустите расширение в Extension Development Host.

Основные команды:

| Команда                    | Назначение                                                   |
| -------------------------- | ------------------------------------------------------------ |
| `npm run compile`          | Собрать extension host, webview, sidebar и обработчик файлов |
| `npm run watch`            | Запустить сборку в режиме наблюдения                         |
| `npm run check`            | Выполнить ESLint, проверку TypeScript и Prettier             |
| `npm run lint:fix`         | Исправить доступные ошибки ESLint                            |
| `npm run format`           | Отформатировать проект с помощью Prettier                    |
| `npm run changelog`        | Полностью пересоздать CHANGELOG из истории Git               |
| `npm run changelog:update` | Добавить изменения после последнего Git-тега                 |

## CHANGELOG и релизы

CHANGELOG формируется из [Conventional Commits](https://www.conventionalcommits.org/):

```text
feat: добавить новый виджет
fix: исправить импорт таблицы
docs: обновить README
chore: обновить зависимости
```

Для ручной генерации выполните:

```sh
npm run changelog
```

При выполнении `npm version patch|minor|major` CHANGELOG пересоздаётся автоматически и добавляется в Git index вместе с новой версией.
